Detailed Description

Simple autoloading class for ORM.

Rules followed:

  • If there is no namespace for the class, then it will try to require the file with EXTENSION in both lower and uppercase
  • If it cannot find a file, does nothing (meaning you just add to the autoload stack if you need a more complex loader)
  • If the namespace begins with a package name that is known to the class (see below) then it will attempt to load it from the specified location
  • Case must either match exactly or be all lowercase

Also includes a Zend style loader (AUTOLOAD_STYLE_ZEND) which searches the include path for the class, replacing underscores with "/".

Changes

Note:
Version 2: You now need to explicitly call register() to use this class automatically
Version 2: To make it more flexible, the AutoLoader no longer automatically uses the Configuration class to load package locations.



Packages
To autoload packages, supply the Autoloader object with an associative array, array keys are package names and values are path names.

Usage
The following is the recommended method of boot-strapping the autoloader for flexible-orm use.

In the application configuration, you can include a "packages" group which identifies where the autoloader should look for classes. For example:

 #INI File "my-application.ini"
 [packages]
 MyNamespace  = "/path/to/my/namespace"
 APlugin      = "/path/to/plugins/aplugin"

So if you asked for MyNamespace, it would look in /path/to/my/namespace/User.

 // -- PHP Implementation
 use \ORM;
 use \ORM\Utilities\Configuration;
 
 require 'flexible-orm/AutoLoader.php';
 
 $loader = new AutoLoader();
 $loader->register();
 
 Configuration::Load('my-application.ini');
 $loader->setPackageLocations( Configuration::packages()->toArray() );

For more information see the Autoloader Tutorial.

Member Function Documentation

addIncludePath ( $  path)

Add a path to the PHP include path.

Essentially a shortcut to set_include_path( get_include_path().PATH_SEPARATOR.$path );

If the same path already exists, it will not be added again. If specifying an absolute path (as opposed to a relative one), the path must exist.

Exceptions:
IncludePathDoesNotExistExceptionif an absolute path is given that does not exist.
IncludePathIsNotADirectoryExceptionif an absolute path is given to a file that is not a directory.
Parameters:
string$path
Returns:
string The new PHP include path
