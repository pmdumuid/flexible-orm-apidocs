Detailed Description

Read-only variable access with filters and defaults.

Given an array of values, allows access (read-only) as properties. Can be looped through and used as an array.

Default values can be specified if the variable is not set or doesn't meet the criteria of the rule (see matchesRule()).

Used by the Request class to represent each set of request variables.

matchesRule ( $  rule,
$  value 
)

Test a value against a rule.

Rules can either be functions names that take the value as an argument or a regular expression pattern. It may use inbuilt (such as ctype_digit ), user defined or anonymous functions.



Example: Function 

 $vars = new Variables();
 
 // User Function
 function myTest( $value ) {
      return $value > 32;
 }
 
 $test = $vars->matchesRule( 'myTest', 54 ); // true as 54 > 32
 
 // Anonymous function (closure)
 $test = $vars->matchesRule(function( $value ){
      return $value < 32;
 }, 54 ); // false as 54 > 32
 
 // Built-in function
 $test = $vars->matchesRule( 'ctype_alpha', 'hello' ); // true
 $test = $vars->matchesRule( 'ctype_digit', 'hello' ); // false



Example: REGEX 

 $vars = new Variables();
 
 $matches = $vars->matchesRule( '/^j\w+$/', 'jarrod' ); // true
 $matches = $vars->matchesRule( '/^a\w+$/', 'jarrod' ); // false
Parameters:
string$ruleThe rule to test. Either a function name or REGEX pattern.
mixed$valueThe value to test
Returns:
boolean True if the rule is passed, false otherwise.
