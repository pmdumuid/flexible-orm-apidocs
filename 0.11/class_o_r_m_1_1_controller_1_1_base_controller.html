Detailed Description

Simple controller class for implementing a MVC stack.

An independent class (ie can be used without the ORM functions and vice-versa) to implement the Controller part of an Model-View-Controller stack.

For usage instructions see the controller tutorial.

There are two hooks available, beforeAction() and afterAction(). Overriding these methods allows for controller-wide functionality such as checking for a logged in user.

performAction ( $  action = null)

Perform a controller action.

Compile the templates and return the output. Does not actually output anything itself. All public properties of the controller are assigned to the template.



Usage 

 // Use smarty for templating
 $request     = new Request( $_GET, $_POST );
 $controller  = new MyController( $request, new SmartyTemplate );
 
 // Perform the action defined in $_GET['action'] and echo the output
 echo $controller->performAction();



Options and Defaults

By default, the template that will be requested from the Template class will be 'controller/action'. This can be overriden by setting the $_templateName property in the action (or the controller).

By default, the controller will look for a template called 'layout'. If it exists and the $_useLayout option is true then the template will be called after the contents of the action's template has been fetched. The output of the action's template will be assigned to a variable action_content in the layout template.

If no action name can be found (i.e. none is provided either in the $action parameter nor the 'action' value of the get parameters) then DEFAULT_ACTION is used from the current class.
