Model Validation

Introduction

The ORM contains a basic framework for validating models. Models are validated before the object is saved.

Rules

First you need to set the rules for a valid object. There are 4 things the model needs for validation:

  1. Override the valid() method in your model class
  2. Check each rule you need
  3. Call validationError() when a rule fails
  4. Return TRUE on success or FAIL when not valid
<?php
class Car extends \ORM\ORM_Model {
    public $colour;
    public $doors;
    public $manufacturer;
    
    /*
     * Check if this Car object is valid
     * 
     * Rules:
     *   Doors must be  > 0
     *   Manufacturer must be set
     */
    public function valid() {
        if( $this->doors <= 0 ) {
            $this->validationError('doors', 'must be greater than zero');
        }
        if( strlen($this->manufacturer) == 0 ) {
            $this->validationError('manufacturer', 'must be set');
        }
        
        // Model is valid if there are no error messages
        return count( $this->errorMessages() ) === 0;
    }
}

Checking and Using Errors

Model validation occurs when you attempt to save an object. You can check whether the save was successful (which it won't be if there are errors) by testing the output of save(). The example below shows a very simple error checking process.

<?php
$car            = new Car();
$car->doors     = 3;
$car->colour    = 'red';

if( $car->save() ) {
    // Car is valid
    
} else {
    // Car is NOT valid
    foreach( $car->errorMessages() as $property => $message ) { 
        echo "Error with the $property field - $message \n";
    }
}
Note:
Save may fail for reasons other than validation, such as database configuration problems.
