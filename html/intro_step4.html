Finding & Manipulating A Single Object

There are a number of ways to find a single object from the database. You can search by id (ORM_Model::Find()), a simple comparison (ORM_Model::FindBy()) or a more complex query (using ORM_Model::Find() with an options array).



Options Array

  • where
    A string for the WHERE statement, optionally with PDO placeholders. If placeholders are present, the values array must also exist.
  • values
    Array of values to be passed to PDOStatement->execute(). If "?" placeholders have been used, this is simply an array of those paramenters in order. If the named placeholders have been used (e.g. :colour), then the array should be an associative array where the keys are the placeholders.
  • order
    The SQL ORDER BY value.



Find

Find a single object

May be used two ways: as a simple lookup by primary key, or as a more complex lookup using an array of options.

Usage: Primary key lookup 

 // Find car with id=2
 @car = Car::Find( 2 );

Usage: Array of options 

 $minDoors = 3;
 $brand    = 'Alfa Romeo';

 // Find the first car that is not an Alfa Romeo where doors is greater than 3
 $car      = Car::Find(array(
      'where' => 'doors > ? AND brand NOT LIKE ?',
      'order' => 'colour DESC',
      'values' => array( $minDoors, $brand ),
 ));

For more information about the options array, see Options Array in the Getting Started tutorial.

Foreign Keys

It is possible to fetch an object with related object(s) in a single query. This only works for the model that has the foreign key in it. For more information, on configuring foreign keys, see Define Foreign Keys.

 // Fetch the car with id 1 and include the Owner object
 @car = Car::Find( 1, 'Owner' );
 echo "This car is owned by ", $car->Owner->name;

Save

Save an object to the database

Either creates a new item or updates an existing one. First checks that the object is valid (see valid()). Returns false if not valid and will not attempt to save the object.

Usage: 

<?php
// Update existing record
$car = Car::Find( 3 );
$car->age++;
$car->save();

// Create new record
$car = new Car();
$car->brand = 'Toyota';
$car->age   = 3;
$car->save();

// Assuming the Car model has an "autoincrement" primary key, it should now be
// populated
echo "New car id: ", $car->id();

// It's also possible to create an item with its primary key set
// Be careful though: if the key exists already it will update the existing record
$car = new Car();
$car->id    = 100;
$car->brand = 'Ferrari';
$car->age   = 30;
$car->save();

// Alternative method of creating a model class:
$car = new Car(array(
    'brand' => 'Toyota',
    'age'   => 3,
));

$car->save();
Note:
Changing the primary key on an existing object will create a new record and leave the old one still in the database. If you want to change the primary key, first call delete() then change the key and call save()


Pre- or Post-Processing Hooks

There are 6 pre and post processing hooks available for this method, to allow for special processing to be done before or after saving. They are called in the following order:

  • beforeSave()

    Override this method to perform an action before an object is saved to the database (creating and updating).

    Note:
    This is called before valid(), meaning it is always called when saving
  • beforeUpdate()

    Override this method to perform an action before an existing object is updated in the database.

    Note:
    This is called before valid(), meaning it is always called when updating
  • beforeCreate()

    Override this method to perform an action before a new object is created in the database.

    Note:
    This is called before valid(), meaning it is always called when creating
  • afterUpdate()

    Override this method to perform an action after an existing object is updated in the database. Only called on success.

  • afterCreate()

    Override this method to perform an after before a new object is created in the database. Only called on success. The object will have its $_id field populated.

  • afterSave()

    Override this method to perform an action after an object is saved to the database (creating and updating). Only called on success.



See also:
_update(), _create()
Returns:
boolean True if successful


Delete

Delete a single item from the database

The object will remain populated.

<?php
// Fetch a car and delete it
$car = Car::Find( 4 );
$car->delete();

// will still output the details of car[4]
var_dump($car); 

// Try to fetch it again
$car = Car::Find( 4 );

// will now output FALSE, as there is not database record
var_dump($car);

Finding & Manipulating Many Objects

The same options exist for finding a number of objects as for finding a single one. All the find function names are named starting with FindAll.

 // Increment the age off all cars by 1
 $cars = Car::FindAll();
 $cars->each(function($car){
      $car->age++;
 });

 $cars->save();



FindAll()

Find all matching objects

Same syntax as Find() except will return a ModelCollection of all matched items, rather than just the first matching object.

FindAllBy()

Find all objects where a single field meets the requirements

This would not be called directly ordinarily, instead call the "magic" method as shown below.

Usage: 

<?php
// Find all red cars
$cars = Car::FindAllByColour( 'red' );

// Find all cars with more than 3 doors
$cars = Car::FindAllByDoors( 3, '>' );

// Find all cars with more than 3 doors and include the owners
$cars   = Car::FindAllByDoors( 3, '>', '\ORM\Mock\Owner' );

// We could now create an array of owners
$owners = $cars->map( 'Owner' );

save()

Save each object in the collection

Returns an array of ids matching the objects.

Usage 

 $people = Person::FindAll();

 $people[1]->name = 'changed';
 $people[]        = new Person( array('name' => 'new person' ) );
 $ids             = $people->save();
Returns:
array Of row ids. This may be especially useful when using autoincrement style keys that are generated by the database
