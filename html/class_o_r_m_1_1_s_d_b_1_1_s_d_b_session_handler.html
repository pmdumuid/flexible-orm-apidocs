Detailed Description

Allow sessions to be stored in SDB instead of locally.

The advantage of this is that sessions are then independant of a machine, so they can span a cluster of machines. Similar to using memcached, but does not need to be configured. Will most likely be slightly slower than memcached.

To use, simply call: 

 SDBSessionHandler::Register()
