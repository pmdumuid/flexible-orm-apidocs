Detailed Description

Simple autoloading class for ORM

Rules followed:

  • If there is no namespace for the class, then it will try to require the file with EXTENSION in both lower and uppercase
  • If it cannot find a file, does nothing (meaning you just add to the autoload stack if you need a more complex loader)
  • If the namespace begins with a package name that is recorded in the Configuration (see below) then it will attempt to load it from the specified location
  • Case must either match exactly or be all lowercase

Also includes a Zend style loader (loadZend) which searches the include path for the class, replacing underscores with "/".


Packages

In the application configuration, you can include a "packages" group which identifies where the autoloader should look for classes. For example:

 #INI File
 [packages]
 ORM          = "/path/to/orm"
 MyNamespace  = "/path/to/my/namespace"

This INI file defines that if the root namespace of a class is ORM, then look in /path/to/orm. It will then use further namespaces as folders within this path. For example it would look for ORM\Mock\Car in /path/to/orm/Mock/Car.php (or /path/to/orm/mock/car.php if the former does not exist).

For more information see the Autoloader Tutorial.
