Define Data Structures

If you are starting a new project (or adding new tables), then you have the luxury of creating your own new data structures.

When creating these, ideally you should work to the following standards:

  • Table names should be lowercase and plural
    cars, sheep, giant_lizards
  • Primary keys should be named id
  • Foreign keys should be named (singular model name)_id
    e.g. car_id, sheep_id
Note:
These are not requirements to run the ORM, as you can simply configure it to use other values.

Tutorial Tables

For this tutorial we will assume we have the following tables:

  • A table named cars with primary key id and foreign keys owner_id and brand (link to table car_manufacturers)
  • A table named owners with primary key id and the fields name and age
  • A table named car_manufacturers with primary key name and the field country
