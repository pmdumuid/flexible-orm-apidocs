Define Foreign Keys

Introduction

In the case when two or more database tables are related, the ORM can simplify retrieving related objects. For example, in our tutorial tables defined in Step 1: Define Data Structures the owners and cars tables are related (car has one owner).

Currently the system only automatically deals with the has one relationship, where the table has a foreign key of the related record in another table (e.g. owner_id). In this example, we would be able to call $car->Owner to get an owner object from a Car object (if it was fetched with Owner, see below).

 $car = Car::Find( 2, 'Owner' );
 echo get_class($car->Owner->name);
Note:
To find two or more objects together in this way, all models must be in the same database.

Basic Model Class

If the foreign keys are in the format [model_name]_[model_primary_key] then no further configuration is required.

In our demonstration tables defined in Step 1: Define Data Structures the relationship between Car and Owner meets this standard (i.e. the foreign key field is owner_id. The relationship between Car and Manufacturer however does not, as they are related through the field brand in the cars table. If the brand field was instead named manufacturer_name then no further configuration would be required.

Custom Foreign Key Format

When the table format does not match the standard format (i.e. table_name underscore primary_key), you have to tell the ORM_Model class what the key is.

This is done by defining class constants. The constant name should be FOREIGN_KEY_[uppercase model name].

We can now define the model for our cars table. We'll call the model Car. We only need to specify the foreign key for the Manufacturer class, because the Owner class uses the foreign key "owner_id".

 class Car extends ORM\ORM_Model {
      const FOREIGN_KEY_MANUFACTURER = 'brand';
 }

Using A Model With Foreign Keys

Model objects are not automatically fetched with their related models included to improve performance. If you want the foreign model objects to be included, you must use the findWith parameters.

For example: 

 $car = Car::Find( 2 );
 echo get_class($car->Owner); // Will be null as it wasn't fetched

 $car = Car::Find( 2, 'Owner' );
 echo get_class($car->Owner); // Will echo 'Owner'

 $car = Car::Find( 2, array('Owner', 'Manufacturer') );
 echo get_class($car->Manufacturer); // Will echo 'Manufacturer'

For more information see ORM_Model::Find() and ORM_Model::FindAll().
