static EnforceReadConsistency ( ) [static]

Check if this model should enforce SDB read consistency

By default, reads are inconsistent in AmazonSDB. This is faster for distributed systems, but it means that reads shortly after writes may not return the updated values. By enforcing read consistency, all reads wait for the updates to be completed.

To enforce read consistency, simply set a class constant 

 const ENFORCE_READ_CONSISTENCY = true
Returns:
string Either 'true' or 'false' for use with AmazonSDB
