Detailed Description

A wrapper for the session variables.

Provides way to avoid locked session when multiple PHP pages are requested.

When a web server receives multiple PHP page request from a single user simultaneous, Apache will block each process until the others are finished.

In some cases, the blocking is undesirable, particularly if the session data is used in a read-only manner (i.e. to display results without any update).

Usage

Where an update is performed:

$session = Session::GetSession();
$session->lock();
$session->set("i", 1);
$session->set("j", 2);
$session->set("k", 3);
$session->unlock();

Where data only needs to be retrieved:

$session = Session::GetSession();
$i = $session->get("i");
echo "The value of i is $i";

Loading session, and subsequently locking it.

// Construction results in loading with a default of non-blocking behaviour
$session = Session::GetSession();
// How (and how not) to set variables.
$j = $session->j;
$session->lock();
$i = $session->i;
$session->i = $i + 1; // Good
$session->j = $j + 1; // BAD - because "j" may have been modified by another script!
$session->unlock();

lock ( )

Lock the session for editing.

Supports nesting, i.e. the following will work:

function setNumber() {
$session = Session::GetSession();
$session->lock();
$session->number = 10;
echo $session->unlock() ? '[setNumber] Unlocked' : '[setNumber] Still locked...';
}
$session = Session::GetSession();
$session->lock();
$session->name = 'Tom';
setNumber();
echo $session->unlock() ? '[global] Unlocked' : '[global] Still locked...';
// Results in output
[setNumber] Still locked...[global] Unlocked
See also:
unlock()
Returns:
int The number of locks currently held (i.e. the number of times that unlock() has to be called to unlock the session.
