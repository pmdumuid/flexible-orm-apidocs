Define Model Classes

Basic Model Class

The simplest model class definition (i.e. when the default conditions are met) is simply extending the ORM_Model class.

Assuming you had the tables defined in Step 1: Define Data Structures you could define the model for Owner as:

class Owner extends \ORM\ORM_Model {
// any custom code could be added to enhance the model
}

This would then allow you to fetch and create database objects in a simple Object-Oriented fashion:

$jarrod = new Owner();
$jarrod->name = "Jarrod";
$jarrod->age = 31;
$jarrod->save();
$allTheOldOwners = Owner::FindAllByAge( '70', '>' );
$allTheOldOwners->delete(); // bit harsh maybe

For more details on accessing and modifying your data, see Access your data!

Custom Table Format

When the table format does not match the standard format (i.e. table name plural of model name and primary key "id"), you have to tell the ORM_Model class what those values are.

This is done by defining class constants. The two most important constants are:

  • TABLE
    Defines the table name name
  • PRIMARY_KEY
    Defines the primary key (default: id)
Note:
You can use both, either or neither of these constants when defining a class. Currently only tables with a single primary key are supported.

We can now define the model for our car_manufacturers table. We'll call the model Manufacturer for the purpose of demonstration:

class Manufacturer extends \ORM\ORM_Model {
const TABLE = 'car_manufacturers';
const PRIMARY_KEY = 'name';
}

More Model Options

Most of the time a system will only use a single database, but in the case where a system must use multiple databases, you can still use ORM2.0.

The connection details for each database must be loaded into the Configuration. The following INI File contains the default database group (which is used when no database configuration is specified) and an alternative database named secondDatabase.

[database] #The default database
name = "test_db"
user = "test"
pass = "password"
[secondDatabase]
name = "test2_db"
user = "test2"
pass = "none"

To create a prepared statement on an alternative database, simply call PDOFactory::Get( $sql, 'secondDatabase' ).

To get a model class to use a separate database, simple define the DATABASE constant to match the config group name. For example:

class ExternalJobs extends \ORM\ORM_Model {
const DATABASE = 'secondDatabase';
}

You can then access this model from a different database in the same way you would normally.

Note:
If you are using $findWith (eg in ORM_Model::Find()) then all the requested models must be stored in the same database.

More Model Options

There are other options for model classes, such as caching and using the AmazonSDB service. For more information see Advanced Features: Special Model Types.
