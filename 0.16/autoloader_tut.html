Autoloader - Managing Packages

Introduction

The AutoLoader class simplifies this including of files, especially when using namespaces. The basic rules for the AutoLoader are:

  • Nested namespaces are treated as subfolders, and are case-sensitive
  • Classes are stored with the extension '.php'
  • You can define additional autoloaders using spl_autoload
  • The package ORM is defined by default as the location of the AutoLoader class but this can be overriden by defining it in the packages array


Packages

The AutoLoader class uses the idea of packages. That is that the root namespace of a class is the package name. For example, the class ORM\Utilities\Configuration is in the package ORM whilst Helpdesk\Models\Issue is in the Helpdesk package. The AutoLoader will then use this package to determine where the class will be.

To define a list of packages, you need an array where the array keys are package names and the values are package locations.

Example

$autoloader = new AutoLoader(array(
'Mynamespace' => '/path/to/my/namespace',
'Helpdesk' => '/MyApps/helpdesk'
));
$autoloader->register();


Packages using Configuration

You can simply use the supplied Configuration class to load packages

To define the location of packages, simply add a [packages] group to the INI file and define the package names and locations. For example, the following would define where the ORM and Helpdesk packages are located.

[packages]
MyNamespace = "/path/to/my/namespace"
Helpdesk = "/MyApps/helpdesk"

This defines the root folder for all classes within this namespace. If an unknown package is requested or an un-namespaced class is called, then the current PHP include path is searched.

With the above INI file, a call to class Helpdesk\Monitor would look for the file /MyApps/helpdesk/Monitor.php

// -- PHP Implementation
use \ORM;
use \ORM\Utilities\Configuration;
require 'flexible-orm/AutoLoader.php';
$loader = new AutoLoader();
$loader->register();
Configuration::Load('my-application.ini');
$loader->setPackageLocations( Configuration::packages()->toArray() );


Nested Namespaces

Further namespaces after the root are treated as subfolders. This means that in the previous example, Helpdesk\Models\Issue would look for the file /MyApps/helpdesk/Models/Issue.php
